1. INTRODUCTION 
 
1.1 Purpose 
The goal of the 'Rat Race Escape' is to deploy a fun, entertaining, and reachable 2D 
asymmetrical multiplayer game. This paper outlines the system requirements and specifications 
of the web application RAT race escape. 
 
1.2 Scope 
The web application will enable the connection of 2-5 players and play a game of cat and mouse 
in real time. The play is centered on one player (the Cat) preying on the rest (the Rats), which in 
turn have to cooperate to find the solution and run. The application does user authentication, 
real-time syncing of the game state, and offers a smooth, browser-based interface, without 
downloads. 
 
1.3 Overview 
The other parts of this paper discuss the functional and non-functional set of requirements in 
creating the Rat Race Escape game. Section 2 describes the functional requirements as viewed 
by the user and Section 3 contains the description of the non-functional requirements.

3. SYSTEM ARCHITECTURE 
 
The system follows a client-server model to ensure fair and synchronized gameplay. 
 
Client-Side (Frontend): Built with Phaser.js, HTML, CSS, and JavaScript. Responsible 
for rendering graphics, playing audio, capturing user input, and communicating with the 
server. 
 
Server-Side (Backend): Built with Node.js. Acts as the authoritative host, managing game 
logic, state, player authentication, and room management. 
 
Communication: A persistent, full-duplex WebSocket connection is maintained between 
each client and the server for real-time, low-latency data exchange using JSON messages. 
 
Hosting: The server can be deployed on a dedicated Ubuntu instance or a cloud platform 
like Render.
